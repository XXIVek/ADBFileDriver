# Драйвер ADB для 1С — Документация

## Изменения от 06.07.2026 — Исправление зависания при запуске ADB сервера

### Проблема
Метод `ПеречислитьУстройства()` зависал в ожидании ответа после запуска ADB сервера. Причина: 
1. `DisconnectDevice()` убивал ADB сервер командой `kill-server`
2. При следующем вызове `EnumerateMtpDevices()` сервер запускался заново
3. Запуск сервера через `AdbExec(L"start-server", ...)` без таймаута мог блокироваться
4. Цикл зависал потому что сервер убивался и запускался снова

### Решение

#### 1. Новая функция `AdbCheckServerStatus()`
- Быстрая проверка статуса сервера через `adb get-state` с таймаутом 2 секунды
- Устанавливает флаг `g_adbServerRunning` при успешной проверке
- Подробное логирование в DebugView и файл лога

#### 2. Новая функция `AdbEnsureServerStarted()`
- Циклическая проверка запуска сервера с таймаутом до 30 секунд
- Алгоритм:
  1. Если сервер уже запущен (`g_adbServerRunning == true`) — проверяем статус
  2. Если adb.exe не найден — возвращаем false
  3. Проверяем текущий статус через `get-state`
  4. Если сервер не запущен — запускаем `start-server`
  5. Ждём и проверяем статус каждую секунду (до 30 раз)
  6. Если `start-server` вернул успех — разрешаем работу даже без подтверждения
  7. При неудаче — одна повторная попытка

#### 3. Изменён `DisconnectDevice()`
- **Удалён вызов `kill-server`** — сервер больше не убивается при отключении
- Сервер остаётся активным между вызовами методов
- Это предотвращает необходимость перезапуска при каждом вызове `ПеречислитьУстройства()`

#### 4. Обновлён `EnumerateMtpDevices()`
- Добавлен вызов `AdbEnsureServerStarted()` в начале метода
- Подробное логирование каждого шага:
  - `[DBG] EnumerateDevices: ENTER — начало ПеречислитьУстройства`
  - `[DBG] EnumerateDevices: Вызов AdbEnsureServerStarted`
  - `[DBG] EnumerateDevices: ADB сервер запущен/проверен успешно`
  - `[DBG] EnumerateDevices: adb.exe найден, путь: ...`
  - `[DBG] EnumerateDevices: Вызов adb devices -l`
- Таймер измерения времени выполнения `adb devices -l`
- Таймаут уменьшен с 60с до 15с

#### 5. Обновлён `SetPropVal` для свойства `КаталогDLL`
- Использует `AdbEnsureServerStarted()` вместо простого `start-server`
- Всегда гарантирует запуск сервера при установке каталога DLL

## Изменения от 07.07.2026 — Исправление Этапа 1

### Проблема
Метод `ПеречислитьУстройства()` зависал в ожидании ответа от ADB.

### Причина
1. ADB выводит вывод в stdout/stderr, но pipe не читался правильно
2. Поиск устройства не работал из-за пробелов вместо `\t` в выводе
3. JSON Serial/Name содержал дублирующиеся символы

### Решение

#### 1. Упрощена логика запуска ADB сервера
- **Убраны** функции `AdbEnsureServerStarted()` и `AdbCheckServerStatus()`
- **Убран** отдельный запуск `start-server` — ADB сам запускает сервер при первой команде
- **Убрано** убивание сервера в `DisconnectDevice()` — теперь сервер завершается через `kill-server`
- **Таймауты убраны** — используется `INFINITE` (без таймаута)

#### 2. Улучшено чтение stdout/stderr
- `PeekNamedPipe` для проверки наличия данных в pipe
- Чтение stdout и stderr по отдельности
- Подробное логирование:
  - `[ADB] Reading stdout...`
  - `[ADB] stdout bytesRead: N`
  - `[ADB] Reading stderr...`
  - `[ADB] stderr bytesRead: N`
  - `[ADB] Process exit code: N`

#### 3. Исправлен поиск устройства
- ADB выводит строки с пробелами вместо `\t`
- Новый алгоритм:
  1. Ищем серийный номер через `wcsstr`
  2. Пропускаем пробелы/табуляции после него
  3. Проверяем что после них идёт `device` или `offline`

#### 4. Исправлено формирование JSON Serial/Name
- Символы копируются по одному через `wchar_t ch[2]`
- Специальные символы (`"`, `\`) экранируются

#### 5. Добавлено завершение ADB сервера
- При вызове `Отключить()` выполняется `adb kill-server`
- Сервер корректно завершается после использования

## Изменения от 08.07.2026 — Исправление Этапа 3 и Этапа 5

### Этап 3 — СписокФайлов(): исправления

#### Проблема 1: Пустые имена файлов
JSON возвращал `{"Name":"","IsFolder":false}` вместо реальных имён.

#### Причина
1. Разделитель строк: ADB возвращает `\n`, а код искал только `\r\n`
2. IsFolder всегда был `false` — не определялся тип файла

#### Решение

##### 1. Нормализация строк
- Добавлена конвертация `\n` → `\r\n` перед разбором
- Пропуск строки `total N` — первая строка вывода `ls -l`

##### 2. Определение IsFolder через `ls -l`
- Используется команда `ls -l` вместо `ls -1`
- Формат вывода: `drwxrwx--- 2 root everybody 3488 2022-08-12 10:12 Browser`
- Первый символ определяет тип:
  - `d` = директория (`"IsFolder":true`)
  - `-` = файл (`"IsFolder":false`)

##### 3. Исправление указателя при копировании JSON
- `dst` после нормализации указывал на конец буфера
- Используется отдельный указатель `jsonDst = fileList + wcslen(fileList)`

##### 4. Исправление GetNParams для СкачатьФайл
- GetNParams case 4 было 2, стало 1 — СкачатьФайл принимает только один параметр

### Проверенные свойства и методы

- `КаталогADB` — каталог расположения ADB (переименовано из "КаталогDLL")
- `ПеречислитьУстройства()` — перечисление ADB устройств ✓
- `Подключить()` — подключение к устройству ✓
- `СписокФайлов()` — список файлов с правильными именами и IsFolder ✓
- `СкачатьФайл()` — скачивание содержимого файла ✓
- `УдалитьФайл()` — удаление файла ✓
- `ЗагрузитьФайл()` — загрузка файла на устройство ✓

### Важное замечание по ЗагрузитьФайл

При выполнении метода `СписокФайлов(Путь)` с указанием пути к несуществующей папке, 
затем методом `ЗагрузитьФайл(ИмяНаУстройстве, Содержание)` создание файла на устройстве, 
**ADB автоматически создаёт необходимую папку и переносит файл**.

Этот алгоритм заложен в ADB — не требуется предварительное создание директорий.
