# Отладка ADBFileDriver

## Проблема 1: DebugView ничего не видит

### Причина
`OutputDebugStringW()` отправляет сообщения в контексте процесса который их вызвал. DebugView должен захватывать сообщения от процесса 1С.

### Решение

1. **Запустите DebugView с правами администратора:**
   - Правый клик на DebugView64.exe → "Запуск от имени администратора"

2. **Настройки DebugView:**
   - `Capture` → `Capture Win32` (должно быть включено)
   - `Capture` → `Capture Global Win32` (включить)
   - `Capture` → `Capture Events` (опционально)
   - `Edit` → `Colors` → убедитесь что цвет сообщений не белый на белом

3. **Очистите буфер:**
   - `Clear` (Ctrl+E)

4. **Запустите 1С:**
   - Откройте 1С и выполните код работы с компонентой

5. **Отфильтруйте сообщения:**
   - `Edit` → `Filter/Hosts...` → вкладка `Hosts` → добавьте `*`
   - В фильтре введите `ADBFileDriver` для показа только сообщений компоненты

### Альтернатива: запись в файл
В DebugView: `Capture` → `Capture Events` → `Save Log` (Ctrl+S)

## Проблема 2: ADBFileDriver.log имеет неверное содержание

### Причина
Файл `ADBFileDriver.log` заполняется только когда:
1. Свойство `ВключитьЛогирование = 1` установлено
2. `ПутьДляФайлаЛогирования` указывает существующую папку

По умолчанию логирование ВЫКЛЮЧЕНО.

### Решение

**В 1С:**
```bsl
МойДрайвер = New('AddIn.ADBFileDriver.ADBFileDriver');

// ВКЛЮЧИТЬ логирование
МойДрайвер.ВключитьЛогирование = 1;
МойДрайвер.ПутьДляФайлаЛогирования = "J:\ADB_Debug.log";

// Теперь все операции будут записываться в файл
Устройства = МойДрайвер.ПеречислитьУстройства();
Результат = МойДрайвер.Подключить("Redmi 9C NFC");
```

**Или установить до подключения:**
```bsl
МойДрайвер = New('AddIn.ADBFileDriver.ADBFileDriver');
МойДрайвер.ВключитьЛогирование = 1;
МойДрайвер.ПутьДляФайлаЛогирования = "%TEMP%\ADBFileDriver.log";

// Теперь вызываем методы
Устройства = МойДрайвер.ПеречислитьУстройства();
```

## Проблема 3: Не подключается к "Redmi 9C NFC"

### Диагностика

**Шаг 1: Проверить EnumerateDevices**
```bsl
МойДрайвер = New('AddIn.ADBFileDriver.ADBFileDriver');
МойДрайвер.ВключитьЛогирование = 1;
МойДрайвер.ПутьДляФайлаЛогирования = "J:\ADB_Debug.log";

Устройства = МойДрайвер.ПеречислитьУстройства();
WriteConsole(Устройства);
WriteConsole(МойДрайвер.КоличествоУстройств);
```

Если `КоличествоУстройств = 0`:
- Устройство НЕ в режиме MTP
- На телефоне: Настройки → Для разработчиков → Режим переноса файлов (File Transfer)
- Или: При подключении USB выбрать "Перенос файлов" / "MTP"

**Шаг 2: Проверить MTP device ID**
В файле `ADBFileDriver.log` или выводе `Устройства` должен быть:
```json
[
  {
    "PnpId": "USB\\VID_2717&PID_FF48&MI_00\\...",
    "MtpId": "SWD\\WPDBUSENUM\\{...}",
    "Name": "Redmi 9C NFC"
  }
]
```

Если `MtpId` пустой — устройство не в режиме MTP.

**Шаг 3: Подключиться по MtpId**
```bsl
// Если Connect("Redmi 9C NFC") возвращает ложь,
// попробуйте подключиться по PnpId:
Результат = МойДрайвер.Подключить("USB\\VID_2717&PID_FF48&MI_00\\...");
WriteConsole(Результат);
WriteConsole(МойДрайвер.Статус);
```

## Проверка режима USB на устройстве

### На Redmi/Android:
1. Настройки → Для разработчиков
2. Убедиться что "Режим отладки по USB" включен
3. При подключении USB: всплывающее окно → выбрать **"Перенос файлов (MTP)"**
4. Или: Настройки → Подключенные устройства → USB → Выбрать **"Файлы / MTP"**

### Проверка в Диспетчере устройств:
1. Открыть Диспетчер устройств
2. Раздел **Портативные устройства** → должно быть "Redmi 9C NFC"
3. Если в **Другие устройства** → драйвер MTP не установлен

### Установка драйверов MTP:
```
Windows Driver Pack for Android MTP
или Samsung Mobile USB Driver (для Samsung)
```

## Этап 1: Тестирование метода ПеречислитьУстройства()

### Проблема
Метод `EnumerateDevices()` возвращает 0 устройств, хотя в Android Studio подключены два устройства:
- A52 пользователя Тома
- Redmi 9C NFC

### Диагностика
Добавлено детальное логирование для выявления проблемы:

1. **Логирование вывода adb devices** (полный вывод):
   - `[ENUM] adb devices FULL OUTPUT:` — полный вывод команды `adb devices -l`
   - `[ENUM] adb line[N]: '...'` — каждая строка по отдельности

2. **Логирование сравнения serial-номеров**:
   - `[ENUM] Device: PnpId=..., Serial=..., ADB=...` — информация об обнаруженном USB устройстве
   - `[ENUM]   Skipping — not in adb devices list` — устройство пропущено
   - `[ENUM]   Checked pattern: '...\tdevice'` — проверенный паттерн
   - `[ENUM]   ADB devices list content:` — список ADB устройств для сравнения
   - `[ENUM]   ADB device[N]: serial='...', full='...'` — каждое ADB устройство с извлечённым serial

3. **Логирование в ConnectToDevice**:
   - `[CONNECT] adb devices FULL OUTPUT:` — полный вывод adb devices
   - `[CONNECT] adb line[N]: '...'` — построчный вывод

### Причина проблемы (предварительная)
Код извлекает serial из PnPId USB устройства (последняя часть после `\`) и ищет его в выводе adb devices. Но ADB может возвращать реальные serial-номера устройств, которые не совпадают с USB-путями Windows.

### Следующие шаги
- Собрать новый лог с детальным выводом
- Сравнить serial-номера из PnPId и из adb devices
- При необходимости скорректировать логику сопоставления
