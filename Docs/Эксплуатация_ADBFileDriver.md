# Драйвер ADB для 1С:Предприятие — Эксплуатационная документация

## Общая информация

**Назначение:** Компонента для работы с Android-устройствами через ADB (Android Debug Bridge) из 1С:Предприятие 8.3.

**Версия:** 2.10.0.0

**Файлы:**
- `1C/Release/Win32/ADBFileDriver_Win32.dll` — 32-битная версия
- `1C/Release/Win64/ADBFileDriver_Win64.dll` — 64-битная версия

---

## Свойства

| Свойство | Описание | Чтение/Запись |
|----------|----------|---------------|
| Версия | Версия компоненты (строка "2.10.0.0") | Чтение |
| ВключитьЛогирование | Включение/выключение логирования (истина/ложь) | Чтение/Запись |
| ПутьДляФайлаЛогирования | Путь к файлу лога (по умолчанию `%TEMP%\ADBFileDriver.log`) | Чтение/Запись |
| Статус | Текущий статус подключения (строка) | Чтение |
| КоличествоУстройств | Количество обнаруженных ADB устройств | Чтение |
| КоличествоФайлов | Количество файлов в текущем каталоге | Чтение |
| ТекущийКаталог | Текущий каталог на устройстве | Чтение |
| КаталогADB | Путь к каталогу с adb.exe | Чтение/Запись |

## Методы

| Метод | Описание | Параметры | Возвращаемое значение |
|-------|----------|-----------|----------------------|
| ПеречислитьУстройства | Перечисляет ADB устройства | Нет | JSON строка с списком устройств |
| Подключить | Подключается к устройству | Имя устройства (Serial) | Boolean |
| Отключить | Отключает от устройства | Нет | Boolean |
| СписокФайлов | Получает список файлов в каталоге | Путь на устройстве | JSON строка |
| СкачатьФайл | Скачивает файл с устройства | Имя файла | Содержимое файла (UTF-16) |
| ЗагрузитьФайл | Загружает файл на устройство | Имя, Содержание | Boolean |
| УдалитьФайл | Удаляет файл на устройстве | Имя файла | Boolean |
| СписокИменФайлов | Получает имена файлов в каталоге | Путь на устройстве | JSON строка |

## Установка

### 1. Подготовка ADB

1. Установите Android SDK Platform-Tools
2. Убедитесь что `adb.exe` доступен в PATH или укажите каталог через свойство `КаталогADB`

### 2. Регистрация компоненты

```
regsvr32 "C:\path\to\ADBFileDriver_Win32.dll"  (для 32-бит)
regsvr32 "C:\path\to\ADBFileDriver_Win64.dll"  (для 64-бит)
```

### 3. Подключение в 1С

```bsl
// Создание объекта
ADB = Новый "ADBFileDriver.ADBFileDriver";

// Проверка версии
Сообщить(ADB.Версия);
```

## Примеры использования

### 1. Подключение и перечисление устройств

```bsl
ADB = Новый "ADBFileDriver.ADBFileDriver";

// Включение логирования
ADB.ВключитьЛогирование = Истина;
ADB.ПутьДляФайлаЛогирования = "C:\Logs\ADB.log";

// Указание каталога с adb.exe (если не в PATH)
ADB.КаталогADB = "C:\Android\platform-tools\";

// Перечисление устройств
JSON = ADB.ПеречислитьУстройства();
Сообщить("Устройств: " + ADB.КоличествоУстройств);
```

### 2. Подключение к устройству

```bsl
// Подключение
Результат = ADB.Подключить(ИмяУстройства);
Если Результат = Истина Тогда
    Сообщить("Статус: " + ADB.Статус);
КонецЕсли;
```

### 3. Работа с файлами

```bsl
// Список файлов
JSON = ADB.СписокФайлов("/sdcard/Download");
Сообщить("Файлов: " + ADB.КоличествоФайлов);

// Скачать файл
Содержание = ADB.СкачатьФайл("test.txt");

// Загрузить файл
Результат = ADB.ЗагрузитьФайл("new.txt", "Содержимое файла");

// Удалить файл
Результат = ADB.УдалитьФайл("old.txt");
```

### 4. Отключение

```bsl
ADB.Отключить();
```

## Важные замечания

### ЗагрузитьФайл
При выполнении метода `ЗагрузитьФайл(ИмяНаУстройстве, Содержание)` для файла в несуществующей папке, **ADB автоматически создаёт необходимую папку**.

### ADB сервер
При вызове метода `Отключить()` ADB сервер завершается командой `adb kill-server`.

### Кодировка
Все файлы передаются в кодировке UTF-8.

## Устранение неполадок

### Устройство не найдено
1. Убедитесь что USB-отладка включена на устройстве
2. Проверьте разрешение на компьютере
3. Выполните `ПеречислитьУстройства()` ещё раз

### Ошибка подключения
1. Проверьте что устройство подключено по USB
2. Убедитесь что adb.exe доступен в PATH или укажите свойство `КаталогADB`
3. Проверьте файл лога для подробной информации (ищите `[FINDADB]`)

### Файлы не находятся
1. Убедитесь что путь указан правильно (абсолютный путь)
2. Проверьте права доступа на устройстве
